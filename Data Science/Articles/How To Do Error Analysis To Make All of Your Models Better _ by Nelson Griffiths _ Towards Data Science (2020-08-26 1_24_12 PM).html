How To Do Error Analysis To Make All of Your Models Better

A step by step guide

Nelson Griffiths
Follow
Oct 16, 2019 · 6 min read

Introduction

There are a lot of ways to improve on out-of-the-box machine learning models. There are things you can do both with the models and the data to obtain better results than you would from the standard scikit-learn version of a model. There is feature engineering, hyper-parameter tuning, and selecting the correct cost function. If we get into deep learning, then we approach the ideas of network architectures, transfer learning, and many others. As a data scientist these are some of the first, and most important, things you learn about building models. Tools like random search and grid search should be in every data scientist’s tool box to help optimize this process and build the best model possible.

When you work on a project most people do all of the things I stated above and then, with either cross-validation or a train test split, obtain some evaluation metric. Those steps are repeated until the best metric score possible is obtained and the model is deemed worthy to use. In no way is this a bad path to take. It usually results in excellent models that provide value in the way they should. With a little extra effort however, you can make every single model you will ever build better. We can accomplish this through error analysis.

Error Analysis

What Is It?

When building a model, you can either go about it randomly or intentionally. A random approach to model building is tempting to take. It requires less effort and still brings about results. A random approach would be when after each iteration of a model, you decide to try something new to make it better without much thought as to why. It is simply this: an idea popped into your head that might improve the F1-score so why not try it? It’s easy and it works.

The intentional approach to building a model is using error analysis. Error analysis requires you to dig into the results of your model after each iteration. You look at the data and predictions on an observational level and form hypotheses as to why your model failed on certain predictions. Then you test your hypothesis by changing the model in a way that might fix that error, and begin the next iteration. Each iteration of modeling becomes more time consuming with error analysis, but the final results are better and will likely arrive faster.

The Right Way To Do Error Analysis

There is not a set way that you should perform error analysis. It is exploratory and it requires some creativity and domain knowledge. That being said, I am going to give you the steps I generally like to take when I do it. Hopefully this gives you a starting point to find great insights from digging into the errors of your model.

Error Analysis Process

With each iteration of the model I start at the beginning of these steps and go through them until I find something that should be fixed. I make my hypothesis, change my model, and start the process again until I am satisfied. In this section I will give you the steps I take, a few things you might find wrong, and some possible fixes for them. We will work from the highest level of data down to the smallest level, searching for errors along the way.

Data Level Errors

This is the type of error analysis that is probably the most common and most people do. I like to start by looking at the model errors on the scale of all of the data.

Over-fitting/Under-fitting

Check if your model is over- or under-fitting. My favorite way of doing this is using learning curves such as the one below.

Generally, I have found in practice that it works best to over-fit first and then use regulation techniques to bring the model back to a good place.

Potential Fixes:

  • If over-fitting then L1 or L2 regularization or more data
  • If under-fitting then a more complex model or more features

Prediction Distribution

Make sure that the distribution of your data looks somewhat similar to the distribution of your predictions. If they don’t look similar then we have some issues.

Potential Fixes:

  • Removing outliers from data
  • Using a different cost function

Group Level Errors

Once the data level errors are taken care of we can look at errors by group. The purpose of this part of the error analysis is to identify groups on which the model performed poorly. Some domain knowledge can be helpful here in when deciding how to create groups. I would suggest beginning with obvious groups that already exist in the data and then move on to creating groups from intuition. For example, I might start by looking at the prediction errors by categorical variables and then move on to splitting numerical variables into bins to examine. When looking at groups, there are a few specific things that I always look out for.

Over/Under Predictions

Check to see if there are any specific groups for which the model over-predicts or under-predicts a large amount of the time. If a model always over-predicts for a certain group, that seems like something we can teach it to fix. Also take note if groups that are performing poorly are lacking data.

Potential Fixes:

  • Penalize over-predictions for that group
  • Oversampling methods if data is lacking
  • Data collection if data is lacking

Large Average Errors

Check to see if any group is contributing way more to the total error than all of the other groups. This is important to note for multiple reasons. First, we obviously want to know what is wrong so it can be fixed. Second, it matters for the interpretability and application of the model. If a model performs well on all but one group, you may just want to use that model for everything except that group if it can’t be fixed.

Potential Fixes:

  • K-Stratified cross-validation to ensure group observations in every split
  • Oversampling methods if data is lacking
  • Data collection is data is lacking

Individual Errors

Now we get down to the nitty-gritty. Once we have done all of the higher level error analysis, it is time to dive in to the errors of each observation. I like to do this by looking at the top 20 errors from my predictions. Unlike the higher level analyses, I do not have any suggestions for common errors you might find. You should go through observations and look at the data and the prediction and determine if the prediction was valid. Sometimes the model made a good prediction and the target for that observation was just wonky. Those are errors we can live with. If you don’t think that the observation seems reasonable, ask yourself why. What information did the model miss that should have helped it make a better prediction? Go through multiple observations looking for patterns of what might be causing the errors. You may find something you missed before that you could fix. You may find that sometimes your model just messes up and there is no reason you can see for it. From this point, you either continue to tweak your model, decide to go get more data, or call your model good.

Implement It

Error analysis takes time and it requires a lot of thinking. Take the time to search for and examine the reasons your model performed poorly. The pattern to good error analysis is this:

  • Find errors
  • Create a hypothesis for what could fix the errors
  • Test hypothesis
  • Repeat

Following these steps with a little creativity and patience will help you build a better model than you could with some hyper-parameter tuning alone.
